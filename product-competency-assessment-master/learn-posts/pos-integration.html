POS·외부솔루션연동 이해하기 — 채널플랫폼과 상품등록간편화

2026-07-19

채널플랫폼이 뭔가요

채널플랫폼은 티오더의 오더서비스와 국내외 POS 파트너사(및 3rd Party 앱) 사이에서, 서로 다른 연동 스펙을 하나로 통합·변환해주는 기반 시스템입니다.

구조는 이렇습니다: 오더서비스 ↔ 채널플랫폼 ↔ POS(국내외 파트너사). POS마다 API가 제각각이라도, 오더서비스 입장에서는 채널플랫폼이 제공하는 단일 API 스펙만 알면 됩니다. 즉 POS사가 바뀌거나 새 POS가 추가돼도 오더서비스 쪽 코드는 바뀔 필요가 없게 만드는 게 이 시스템의 존재 이유입니다.

연동방식 3종 — 매장마다 다릅니다

  • 티오더에이전트: 매장에 설치되는 프로그램으로 POS와 통신합니다.
  • 포스에이전트: POS사가 자체적으로 제공하는 에이전트로 연동합니다.
  • 클라우드 어댑터: 로컬 설치 없이 클라우드에서 직접 연동합니다 (최근 전환이 진행 중인 방식 — 장애 유연성·운영 비용 절감이 목적).

같은 POS 회사라도 선결제·부가세노출·매출연동·상품등록 지원 여부가 다르므로, 특정 매장에서 "왜 이 기능이 안 되지?"라는 문의가 오면 먼저 그 매장 POS의 연동방식부터 확인해야 합니다. 상세 비교는 사이트의 POS 탭에서 볼 수 있습니다.

왜 같은 어드민 화면인데 매장마다 다르게 동작하나요 — 포스 기준 모드 vs 어드민 관리 모드

이게 실제로 사내에서 가장 많이 헷갈리는 부분입니다. 모든 매장은 상품·분류(카테고리) 데이터의 "기준"이 어디인지에 따라 아래 두 모드 중 하나로 동작합니다.

  • 어드민 관리 모드 (v0.8): 상품/분류 데이터의 기준이 티오더 어드민입니다. 신규 상품이 생기면 어드민에서 사람이 직접 등록해야 티오더 태블릿에 반영됩니다.
  • 포스 기준 모드 (v1, = "상품등록간편화"): 상품/분류 데이터의 기준이 POS입니다. POS에만 상품을 등록하면 자동으로 티오더 태블릿까지 동기화됩니다 — 어드민에 다시 입력할 필요가 없어서 "간편화"라고 부릅니다. 목적은 이중 입력에 따른 CS 비용 감소입니다.

즉, POS의 상품/분류 체계와 티오더 태블릿의 상품/분류 체계가 항상 같은 시스템이 아닙니다. 어느 쪽이 "원본"인지가 매장마다 다르고, 이걸 모르면 "왜 우리 매장은 되는데 저 매장은 안 되지?" 같은 오해가 CS로 계속 올라옵니다.

Phase 1(포스 기준 모드 최초 배포)의 한계: 포스 기준 모드에서는 티오더 어드민에서 분류를 직접 만들 수 없었습니다. 그래서 광고용 분류를 만들려면 대리점을 통해 별도 요청을 해야 했고, POS에 등록하지 않은 정보성 상품(매장안내, 리뷰 안내 등)도 단독으로 구성할 수 없었습니다.

Phase 2 하이브리드(2026-06-16 배포)로 개선된 점: 포스 기준 모드에서도 어드민에서 분류를 직접 만들 수 있게 됐습니다. 이때 "포스에서 온 분류"와 "어드민에서 직접 만든 분류"를 색으로 구분합니다(포스 분류는 빨간색, 어드민 분류는 보라색). 상품명도 [직접 입력] 버튼으로 POS 동기화와 무관하게 수정할 수 있고, [되돌리기] 버튼을 누르면 다시 POS 기준값으로 즉시 복원됩니다 — 이후에는 다시 POS 동기화를 따라갑니다.

정리하면: 한 매장 안에서도 "포스에서 자동으로 오는 것"과 "어드민에서 사람이 직접 얹은 것"이 섞여 있을 수 있다는 걸 아는 게 핵심입니다.
