티오더AI 이해하기 — 사장님을 위한 카카오톡 AI 에이전트

2026-07-19

정의

티오더AI(구 티오더GPT)는 사장님이 카카오톡 채팅만으로 메뉴 품절 처리, 옵션 수정, 이미지 변경, 매출 조회 같은 매장 운영 작업을 처리할 수 있게 해주는 AI 에이전트입니다. 2026-03-31 정식 런칭했습니다.

핵심 동작 원리

1) 발화를 3가지로 분류해서 처리합니다

  • Action: 매장 데이터를 변경하는 실행 요청 (예: "떡볶이 품절 처리해줘")
  • Query: 데이터 조회 요청, 변경 없음 (예: "어제 매출 얼마야?")
  • General: 상담·문의·잡담 등 에이전트 기능 외 발화 (예: "티오더 요금제 알려줘")

2) 매장 데이터를 바꾸는 작업은 실행 전 반드시 확인을 받습니다 (Human-in-the-Loop). 특히 아래 조건에서는 더 신중하게 확인합니다.

  • 영구 삭제 작업 (복구 불가, 예: 분류 삭제)
  • 다중 대상 일괄 변경 (3건 이상, 영향 범위가 큼)
  • 가격·판매 변경 (매출에 직접 영향)
  • 구조 변경 (분류 생성/삭제, 상품-분류 재배치 등 메뉴판 전체에 영향)

반대로 조회(Query)는 데이터 변경이 없으므로 확인 없이 즉시 실행됩니다.

3) 대부분의 변경은 되돌릴(Undo) 수 있습니다 — 단, 전부는 아닙니다.

  • Undo 가능: 상품명 변경, 판매 활성/비활성, 품절 설정/해제, 이미지 변경(7일 이내), 분류명 변경, 분류 판매 설정 (그 외 대부분은 30일 이내)
  • Undo 불가: 상품 설명·알레르기 표시, 주문수량 제한, 맵기 단계, 스티커/태그, 할인가, 상품 순서·분류 이동, 분류 생성/삭제, 옵션 그룹 편집, 신규 상품 등록, 가격 변경, 영구 삭제된 상품/분류

4) 처리 범위 밖 문의는 상담사(세일즈포스 매트릭스챗)로 자동 이관됩니다 — 에이전트 처리 범위 외(CS·계약·장애 등), 사용자가 "상담사 연결"을 명시적으로 요청, 또는 같은 요청이 3회 이상 실패한 경우입니다.

실제로 가장 많이 쓰이는 기능 (호출 건수 기준 Top 10)

  1. 상품목록 조회 (25.3%) — 사장님이 상품명을 정확히 모를 때 전체 조회부터 시작하는 패턴
  2. 이미지 변경 (10.8%)
  3. 분류 리스트 조회 (8.7%)
  4. 분류에 상품 등록 (5.6%)
  5. 상담사 이관 (5.1%)
  6. 상품 추가 가이드 (3.4%)
  7. 상품명 변경 (2.9%)
  8. 지식베이스 검색 (2.8%)
  9. 상품 상세 조회 (2.5%)
  10. 옵션 목록 조회 (2.5%)

작업 유형별로 보면 이미지/사진 변경(19.7%), 옵션 수정(14.4%), 상품명 변경(8.4%), 분류 관리(5.6%), 배너 관리(4.7%), 품절/상태 변경(4.7%) 순으로 많이 요청됩니다.

실제 쓰이는 규모 (Day1~81, 2026-03-31~06-19 누적)

  • 이용 매장 4,628개, 이용 유저 5,544명, 처리 작업 42,812건
  • 재방문율(2일 이상) 55.4%, 재이용 평균 간격 8.1일(중앙값 3일) — 주 1~2회 정기 이용 패턴
  • 상담사 이관율은 초기(신규 가입 폭증 시기) 20%대까지 올랐다가 8.5% 수준으로 안정화
  • "N개 처리했다고 답했지만 실제로는 일부만 처리된" 오류(할루시네이션)는 초기 10.8%에서 현재 0.9%까지 억제
  • 이용 피크는 17~18시(저녁 영업 준비 전), 금요일이 가장 많음

최근 업데이트
