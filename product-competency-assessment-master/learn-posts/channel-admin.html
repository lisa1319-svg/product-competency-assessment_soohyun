어드민 이해하기 — 채널관리자 (파트너사 대상) ※ 범위 한정

2026-07-19

이 글은 "채널관리자"(Channel Admin) 기준입니다. 사장님이 매장을 직접 관리하는 티오더관리자와는 다른 어드민입니다 — 사내 컨플루언스에 티오더관리자 문서가 아직 비어 있어 이 부분은 추후 별도로 보강해야 합니다.

정의와 배경

채널관리자는 채널플랫폼의 운영·관리를 위해 프로덕트와 파트너사를 관리하는 어드민입니다. (구)미들웨어 어드민을 대체했습니다. 미들웨어 어드민의 비효율적인 프로세스(특히 매장 신규 등록 과정)를 재개발해서, 채널플랫폼 운영에 필요한 시간·인적 비용을 줄이는 게 목적입니다.

누가 쓰나요

  • 내부 사용자: 서비스 운영/테스트가 필요한 운영·QA 담당자, 개발 관련 정보·히스토리 파악이 필요한 개발 담당자, 비즈니스 현황·확장 인사이트가 필요한 실무자·경영진
  • 외부 사용자: POS/VAN/PG사의 개발·운영·영업 카운터파트, SaaS 고객사(화이트라벨링)의 개발·운영·영업 카운터파트, 현장 설치·CS 처리를 담당하는 티오더 외주 직원

주요 기능

1) 채널플랫폼 연동 관리

  • 연동 등록: 티오더 어드민에서 매장 등록 시 채널 API 키 발행 자동화, 자동화 실패 시 수동 등록 지원
  • 매장별 연동 관리: 매장 연동 현황 조회·수정, 연동 활성화 관리(on/off)

2) 권한 관리(외부 사용자) — 내부/외부 사용자 권한을 조회·수정·검색

3) 연동 개발 관리 — 파트너사별 개발 문서, 담당자, API Access Key, 연동 구축 가이드를 등록·조회·수정·검색

4) 티오더 에이전트 관리 — 티오더 에이전트 설치파일을 등록·조회·수정·삭제

5) 대시보드 — 운영에 필요한 매장·지역·시간대별 현황/히스토리, 장애 대응·개발에 필요한 에러 트래킹 지표 제공

왜 이 어드민이 있는지 물어보면

"매장 신규 등록·연동 관리 과정을 시스템으로 전환해서, POS/파트너사와의 연동 정보를 사람에게 의존하지 않고 관리하기 위해서"라고 답하면 됩니다. 이전에는 매장 신규 등록, 파트너사 정보 관리가 전부 사람이 수작업으로 처리하던 영역이었고, 이게 CS·개발 대응 시간을 늘리는 원인이었습니다.

참고 — 최근 로드맵 흐름 (2025년 기준)

  • 실시간 로그 모니터링, 미들웨어 어드민 셧다운(국내/해외 리전 분리 구축)
  • 컨트롤타워 기능 내재화(관리기능 내재화, 이력관리, 알림 메시지 관리)
  • 설치 파일 업데이트 관리, 파일 다운로드 간소화(구글드라이브 활용)
  • 매장관리 > POS 원격접속 기능 추가, 대시보드 구조화
